# Workspace Mapping Policy v7

## Controller Workspace
- `openclaw/` is the top-level controller workspace.

## Role Workspaces
- each agent under `openclaw/agents/<role>/` is a standalone role workspace candidate.

## Recommended Use
- strategic requests begin at controller
- role-specific deep work happens in role workspace
- reviewer work should remain isolated from internal execution tracks
- reproducibility audit should remain isolated from advocacy roles
