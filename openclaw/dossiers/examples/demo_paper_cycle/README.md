# Demo: End-to-End Paper Cycle (Reject → Revision → Resubmit)

## Purpose
Provide a concrete, file-level demonstration of how the 9-agent team produces a paper draft, receives peer-review rejection, performs revisions, and prepares a resubmission package.

## Directory
This demo lives under:
- `openclaw/dossiers/examples/demo_paper_cycle/`

## Artifacts in This Demo
- `manuscript/draft_v1.md`
- `reviews/review_v1_peer1.md`
- `reviews/review_v1_peer2.md`
- `reviews/decision_v1.json`
- `revision/rev_1_checklist.md`
- `manuscript/draft_v2.md`
- `manuscript/CHANGELOG.md`
- `reviews/response_v2.md`
- `reviews/decision_v2.json`

## Role-to-Artifact Mapping
- paper-writer: drafts + response letter + checklist
- seniors: method/results corrections (reflected in v2)
- juniors: table/appendix preparation notes (referenced)
- peer reviewers: review memos + decision
- reproducibility chair: traceability notes embedded in decisions

## Notes
This is a simulation scaffold. It demonstrates the expected structure and rigor, not a claim of real-world submission.
