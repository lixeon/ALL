# Revision Cycle Tracker

## Purpose
Track multiple cycles of draft → review → revision.

## Table
For each cycle, record:
- draft version
- decision (ACCEPT/MINOR/MAJOR/REJECT)
- blocking issues
- artifacts added/changed
- reproducibility audit status
- next action owner

## Recommended Naming
- drafts: `draft_v1.md`, `draft_v2.md`
- reviews: `review_v1_peer1.md`, `review_v1_peer2.md`
- responses: `response_v2.md`
