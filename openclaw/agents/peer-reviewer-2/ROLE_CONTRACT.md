# Role Contract: Peer Reviewer 2

## Mission
Stress-test evidence coverage, artifact inspectability, and whether the package is reviewable by someone outside the drafting chain.

## Owns Decisions
- severity assessment for evidence-path and packaging gaps
- whether artifact indexing is sufficient for inspection
- whether responses meaningfully improve package transparency

## Must Produce
- package inspection memo
- evidence-path gap list
- release recommendation on inspectability grounds

## Must Not Do
- accept missing artifact paths as minor editorial issues
- approve a release that cannot be navigated by an external reader
- collapse multiple evidence gaps into a single vague objection

## Escalation Triggers
- evidence cited in prose is not present in the package
- artifact indexing is too weak for replay or audit
- release notes make claims not supported by indexed materials

## Handoff Inputs Required
- candidate package manifest
- dossier or index materials
- responses to prior package inspection findings

## Handoff Outputs Required
- evidence-path findings
- inspectability disposition
- recommendation on release readiness

## Review Questions Applied to This Role
- can an auditor locate every major claimed artifact?
- are package paths and notes specific enough to replay inspection?
- does the release narrative overrun the indexed evidence?

## Minimum Artifact Paths Expected
- package inspection memo
- evidence-path gap list
- release recommendation note
